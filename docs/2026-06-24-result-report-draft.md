# 프로젝트 결과보고서 (초안)

> WE-Meet 결과보고서용 초안. **굵게 [팀작성]** 표시는 팀이 채울 곳(소감·사진·멘토 코멘트). 수치는 v0.x 실측(`output/`).
> 과제명(권장): **Shortcut을 고려한 보정형 네트워크 침입 탐지·Triage — 날짜 교차 평가와 SHAP 근거 설명**

## 1. 프로젝트 개요
### 1.1 과제명
AI 기반 공격탐지 에이전트 — 네트워크 침입 탐지 및 분석가용 Triage (CSE-CIC-IDS2018 기반).
### 1.2 배경 및 필요성
- 네트워크 공격(포트스캔·DoS/DDoS·브루트포스·봇넷 등)은 트래픽에 흔적을 남기지만, **규칙 기반 탐지는 신규·변형 공격에 약하고** 사람이 로그를 직접 보기엔 양이 많다.
- ML 기반 탐지가 대안이나, 공개 벤치마크(CSE-CIC-IDS2018)는 **라벨 오류·"지름길(shortcut)" 특징**이 알려져 있어, 무작위 분할의 높은 정확도가 실제 성능을 과대평가하는 함정이 있다.
- 따라서 단순 "정확도 자랑"이 아니라 **(a) 과대평가를 피하는 정직한 평가, (b) 분석가가 신뢰·해석할 수 있는 위험도·근거 제공**이 필요하다.

## 2. 목표 및 수행 내용
### 2.1 팀 구성 및 역할
| 이름 | 역할 |
|---|---|
| 팀장(20220****) | 데이터 전처리, 탐지 모델·평가, conformal triage, 보고서 |
| 팀원(20230****) | 에이전트 구조, SHAP·설명·대시보드, 통합 테스트 |

### 2.2 해결할 문제 및 목표
정상/공격 **이진 분류** + **0~100 위험점수 + SHAP 근거 + 자연어 설명**을 한 시스템으로 통합. **원칙: 탐지·점수는 ML이 결정, LLM은 설명만**(재현성·프롬프트 인젝션 방지).

### 2.3 문제 해결 과정 (핵심 서사 — "발견과 극복"의 연속)
1. **베이스라인(v0.1):** 1일치(BruteForce)로 RF/XGB → F1 **1.000**. 그러나 **단일 피처 `Fwd Seg Size Min`이 중요도 99.2%** = 지름길로 푼 허수임을 발견.
2. **정직한 평가로 전환(v0.2):** 다중 일자 결합 + 누수/지름길 피처 제거(F0/F1/F2) + **날짜 교차 평가**(과거 일자로 학습→미래 일자 테스트) + 테스트는 풀 자연분포. → **무작위 0.99 vs 날짜교차 대부분 0**(unseen 일반화 붕괴) 확인.
3. **보정·Triage(v0.4):** 글로벌 확률 보정이 일자별 분포 변화로 깨짐 → **conformal benign-tail p-value + Neyman-Pearson 임계값**(분포무관 오탐 제어).
4. **병목 규명 — "표현 한계 아니라 전이 문제":** target-day function oracle에서 Bot 0.996(그날 안에선 거의 완벽 구분) → **F2에 정보는 충분, 문제는 과거→미래 전이(conditional shift)**임을 입증. (이상탐지·Deep SAD는 unseen에 실패 확인)
5. **temporal 피처로 극복(핵심 성과):** 원본 Timestamp로 Dst Port별 causal 시간창(1~600초)·반복 규칙성·연결실패율 추가 → **unseen recall 급상승(Bot 0.01→0.49 등).**
6. **근거·설명·대시보드(v0.3/v0.5/v0.6):** SHAP 근거 + LLM 자연어 설명(판정 불변) + Streamlit triage.
7. **다층 방어 관점 + 보안로그 레이어:** 단일 NDR 한계를 현업(SIEM+NDR+EDR 다층) 맥락에 위치시키고, Windows 이벤트로그 분석 보조 레이어 추가.

## 3. 수행 결과
### 3.1 데이터
CSE-CIC-IDS2018(dhoogla 정제판) 10일치, 666만 flow, 78컬럼(F2 robust: 식별자·Port·Protocol·fingerprint 피처 제거). 동결 manifest: `docs/2026-06-23-data-experiment-freeze.md`.
*한계: 정제판은 cleaned이며 라벨 교정판은 아님 → 알려진 라벨 오류(Infiltration·웹 BF) 명시, Infiltration은 헤드라인 제외.*

### 3.2 이진 탐지 — 전 공격군 날짜 교차 (XGBoost/F2, conformal NP 임계값)
시간순(테스트일 이전 clean 일자로만 학습), 테스트=풀 자연분포. `output/metrics_allfolds.json`.
| 테스트일(공격) | 노출 | AP | rec@20FP | rec@100FP(실제FP) | 판정 |
|---|---|---|---|---|---|
| DoS GoldenEye/Slowloris (2/15) | unseen | 0.46 | 0.02 | 0.03 | ❌ |
| DoS Hulk (2/16) | same-family | 0.17 | 0.41 | 0.55 (**FP 91%**) | ❌ 오탐 폭발 |
| DDoS LOIC-HTTP (2/20) | unseen | 0.98 | 0.0 | 0.0 | ⚠️ 랭킹↑ 운영 0 |
| **DDoS HOIC (2/21)** | same-family | **1.00** | **0.825** | 0.825 | ✅ 유일 작동 |
| Web BF/XSS/SQLi (2/22) | unseen | 0.002 | 0.0 | 0.01 | ❌ |
| Web (2/23) | same-family | 0.35 | 0.41 | 0.43 | ⚠️ 부분 |
| Bot (3/2) | unseen | 0.31 | 0.0 | 0.0 | ❌ |
| Infiltration (2/28·3/1) [라벨불신] | 부록 | 0.13~0.26 | 0.003 | 0.005~0.008 | ❌ (부록) |
- **핵심 발견:** 날짜 교차에서 **운영 가능한 건 DDoS-HOIC 하나뿐.** "same-family면 된다"는 가설도 깨짐(DoS Hulk는 같은 군 학습에도 FP 91%). **AP 높은데 운영 recall 0**(LOIC-HTTP·Bot) = AP가 운영가치 과대평가. 무작위분할 F1 0.99는 함정.
- 운영 recall은 보정 benign 크기에 민감(DDoS cap20k 0.82 → near-full 0.35). 확률 품질 BSS: DDoS 0.82 양호 / Web·Bot 음수 → 확률 출력 보류.

### 3.3 공격 유형 분류 (multiclass, 6클래스) `output/metrics_multiclass.json`
| 평가 | macro-F1 | 비고 |
|---|---|---|
| (A) 무작위분할 상한 | **0.945** | Benign/Bot/BF/DoS/DDoS 0.98+, Web 0.70(희소) |
| (B) 시간순(앞→뒤 일자) | **0.219** | 공격군이 시간상 안 겹쳐 일반화 붕괴 |
- 이진과 동일한 결론: 무작위 고성능은 환상, 시간/공격군 변화에 취약.
- 산출물: `output/metrics_v0{1,2,4b}.json`.

### 3.3b ★ temporal 피처로 unseen 일반화 극복 (핵심 성과) `output/metrics_temporal2.json`
원본 Timestamp 기반 Dst Port별 causal 시간맥락 피처(시간창 1/10/60/300/600초 flow수·SYN/RST·연결실패율·반복 규칙성) 추가. 운영 임계값(worst-day, 실제 FP ~15~18/100k):
| 공격(unseen) | F2(시간맥락 無) | **+temporal** | 비고 |
|---|---|---|---|
| **Bot** | 0.009 | **0.49** | 봇넷 반복연결(beaconing)을 시간맥락이 포착 |
| **DoS** | 0.51 | **0.62** | |
| **DDoS** | 0.38 | **0.50** | |
| **BruteForce** | 0.74 | **0.98** | 반복 인증시도 |
| Web | 0.02 | **0** | flow 희소+payload 없음=구조적 한계 |
- **핵심:** 병목은 모델 용량이 아니라 **데이터에 시간맥락이 없던 것**. 추가하니 unseen이 0에서 급상승. (운영=과거기반 임계값에서도 유지, 오라클 착시 아님)
- 공격군별 최적 피처셋이 다름(Bot=장기창 포함, DoS/BF=단기창). IP 부재로 host/edge 피처는 불가(공개데이터 IP 제거) = Bot 0.49가 단일 데이터셋·IP없음 천장.

### 3.6 보안로그 분석 레이어 (다층 방어) `output/log_analysis.json`
NDR(네트워크)이 못 보는 호스트 내부를 보완. Bot 일자 Windows 이벤트로그(.evtx) 파싱 → 호스트별 EventID 분포·보안이벤트(7045 서비스설치/7040 변경 = 봇넷 지속성 흔적)·드문 이벤트 이상징후.
- 한계: System 채널 위주·공격 라벨 없음 → 분류 아닌 **이상징후 분석 데모**. 다층 방어의 "로그 레이어" 개념 실증.

### 3.4 탐지 근거 (SHAP)
전역 주도 피처: `Fwd/Bwd Packet Length Max`, `Flow/Fwd IAT Min`(플러드 신호), `Fwd Header Length`. 개별 이벤트별 top-5 기여 피처 + 정상 중앙값 대비 해석. `output/shap/`.

### 3.5 위험점수·Triage·설명·대시보드
- 위험점수 = conformal benign-tail 운영점수(0~100), 우선순위 P1~P3(과거 FP예산 임계값).
- 설명: 불변 JSON → 5섹션(요약/근거/위험도/권고/주의), 판정·점수 불변, 미탐/오탐 주의 명시.
- Streamlit 대시보드: 위험순 이벤트·SHAP 근거·설명·성능. `src/app.py`.

## 4. 활용 방안 및 기대 효과
### 4.1 기존 방식 대비 차별성
| 기존 | 본 프로젝트 |
|---|---|
| 무작위 분할 고정확도 보고 | **날짜 교차 + shortcut 감사**로 과대평가 제거 |
| 단일 정확도 | **계층별(전이/희소/unseen) 정직한 한계 제시** |
| LLM이 탐지 판정 | **LLM은 설명만**(비권한·인젝션 면역) |
| 임계값 0.5 | **과거기반 FP예산 + 분포무관(NP/conformal)** |
### 4.2 기대 효과
분석가가 신뢰 가능한 위험도·근거로 의심 이벤트를 우선 처리(triage). 평가 프로토콜·코드는 후속 보안 실습·연구에 재사용. **무엇보다 "단일 ML로 다 못 잡는다"를 정직하게 실증** → 현업의 다층 방어(SIEM+NDR+EDR) 필요성을 데이터로 이해.
### 4.3 핵심 결론 (1차)
- **AI 공격탐지는 무작위 평가에선 99%지만, 정직한 날짜교차에선 대부분 0.** 원인은 표현 부족이 아니라 **전이(날짜·공격군 변화)**임을 oracle로 입증.
- **시간맥락(temporal) 피처로 unseen을 0→0.49(Bot)까지 끌어올림** — 모델이 아니라 데이터 표현이 열쇠.
- 단일 NDR로는 한계 명확 → **현업이 다층을 쓰는 이유를 우리 손으로 규명.** (Sommer&Paxson 2010 / base rate fallacy와 일치)
### 4.4 한계 및 향후
- **Web(XSS/SQLi)**: flow에 payload 정보 없어 구조적 미탐 → 패킷/HTTP 레이어 필요.
- **IP 부재**: 공개 데이터가 IP 제거 → host/edge(beaconing) 피처 불가. 원본 PCAP(440GB) 재처리 시 Bot 추가 상승 기대.
- **다층화**: NDR + 로그 + EDR 통합, LLM 에이전트 오케스트레이션이 다음 단계.
- 현재 결과는 탐색 라운드(테스트 관측됨) → blind 재실행으로 확정 필요.
