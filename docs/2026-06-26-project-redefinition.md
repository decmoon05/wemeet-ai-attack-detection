# 프로젝트 재정의 — 교수 1차 피드백 반영 (2026-06-26)

## 교수 피드백 요지
1. 데이터셋(CSE-CIC-IDS2018) 선택은 적절.
2. **"분석 프로젝트인지, 탐지 강화(정확도 향상) 실험 프로젝트인지 명확히 하고 구체화하라"** — 어떤 모델·방법, 시뮬레이션/평가 환경 구성, LLM 기반 탐지 세부 등.
3. **강화라면 특정 공격으로 범위를 좁혀 정확도를 높여보라.**
4. 추천 논문 2편: Wang et al. 2023(딥러닝 IDS 벤치마크), ROSE-BOX 2025(경량 IDS).

## 확정된 정체성 (2026-06-26 결정)
**"봇넷(Bot) 탐지 강화 실험 프로젝트 — 날짜교차 정직평가 위에서 경량 ML 탐지기로 미관측 봇넷 탐지율을 끌어올린다."**
- 유형: **탐지 강화** (특정 공격 = Bot으로 범위 축소) + **정직 평가**를 차별점으로 결합.
- 차별 한 문장: *추천 논문들은 쉬운 DoS에서 무작위 분할로 100%를 내지만, 우리는 어려운 봇넷을 날짜교차(과거→미래) 정직평가 위에서 경량 트리 파이프라인으로 0.009→0.49(추가 개선 목표)까지 올린다.*

## 왜 Bot인가
- temporal 피처로 이미 **0.009 → 0.49** 개선(스토리 연속).
- 추천 논문들이 안 다룬 어려운 미관측 공격(Wang: Bot은 잘 잡히나 무작위 분할 기준 / 날짜교차에선 어려움). 표본 14.5만으로 충분(Web·Infiltration의 표본 부족 문제 없음).

## 구체화 (교수 피드백 항목별 답)
### 1. 모델·방법
- 베이스: **XGBoost(트리)**. 딥러닝 미채택 근거 = Wang 2023에서 트리 대비 딥러닝 이득 미미 + ROSE-BOX도 트리 기반.
- **ROSE-BOX 차용:** XGB-assisted RF 피처선택 + EarlyStopping으로 피처 최소화 → 베이지안 최적화로 XGBoost 튜닝(경량·고효율).
- **우리 고유:** ① temporal 피처(Dst Port별 causal 1·10·60·300·600초 시간창 집계, SYN/RST·실패율·규칙성) ② 날짜교차 평가 ③ conformal+Neyman-Pearson 임계값(분포무관 FP 제어).
- SMOTE는 신중히(ROSE-BOX도 과적합 한계 자인) → class_weight/threshold 조정과 비교.

### 2. 시뮬레이션·평가 환경
- 데이터: CSE-CIC-IDS2018. **학습 = Bot 이전 날짜(2/14~3/1), 시험 = 3/2 Bot**(날짜교차, 미관측).
- 환경: Python 3.13 · scikit-learn · XGBoost. 효율 비교 위해 추론시간(μs/sample)·CPU%·**모델 크기(MB)** 측정(ROSE-BOX는 모델크기 미측정 → 우리가 더 나아감).
- 지표: Bot **recall@고정FP** · FPR · AP · 효율 3종. accuracy 단독 금지.

### 3. LLM 역할 — 오해 정정 (중요)
- **탐지·점수·우선순위 = ML(XGBoost)이 결정. LLM은 판정/점수를 못 바꾸고 "설명"만.** 
- 1차 PPT가 "LLM 기반 탐지"로 읽힌 듯 → 다음 발표/회신에서 **"탐지=ML, LLM=설명 전용"**을 명확히 못 박는다. 이유: 재현성·프롬프트 인젝션·환각 차단.

## 목표 (정량)
- Bot 미관측 recall: 현재 0.49(@~18 FP/10만) → **목표 0.6+ 또는 통계적으로 의미 있는 추가 향상**, FPR 통제 하에.
- 경량성: baseline(RF/XGBoost/딥러닝) 대비 추론시간·CPU·모델크기에서 우위 또는 동급 정확도 유지.
