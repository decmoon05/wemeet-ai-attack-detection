# 지도교수 2차 피드백 — 과제 #2(진행간 어려운 점) 답변 (2026-06-28)

> 교수님 출장 중이라 회신이 늦었고, "각 번호당 책 1권 분량이라 간략히 적는다"는 전제로 주신 답. 기업 멘토 매칭 전까지는 교수님이 회신. 다음 주차 전체 과제 제출 시 종합 회신 요청받음.

## 1. (정확도/일반화)
- 알려진 위협 탐지의 일반적 문제. 데이터가 동일하면 정탐율 높은 건 당연. **데이터가 달라지면** 어떤 장비는 정탐율을 유지하고 어떤 건 전혀 못 한다 — 이것이 1번 질문의 답.
- 실무에서는 **데이터의 허점을 공격**한다. 즉 얼마나 충실히 노력했는지, **규칙이 얼마나 정교화됐는지**를 평가한다. 이게 중요.
- 정교화 = 단순 패턴(위협 탐지 규칙)이 아니라, **다양한 상황을 고려해 단위 행위(패킷 등 단일 로그)의 특성을 명확히 분석해 탐지해내는 능력.**

## 2. (처음 보는 공격)
- 맞다. 단, **공개 데이터셋은 모두 재현된 = 알려진 공격이다. 제로데이가 아니다.** (이게 중요)
- 현재의 공격 탐지 기법은 **제로데이까지도** 탐지해낸다.
- 패턴은 특정 문자열·길이 특성만이 아니다. **특히 봇넷은 행위(behavior)에 의한 것**일 수 있다.
- 현실에서 **알려진 공격은 100% 다 잡는 게 기본.** 단 봇넷 등은 **FP 가능성이 높고**, 실효성 고려 시 **FN이 있을 수 있어** 적절한 기법이 요구된다.
- 힌트: **단순 장비(단위 위협 탐지)에 의존하지 않는다. 시나리오 기반 탐지** 등을 목표로 운영한다.
- AI 비중은 너무 다양해 "얼마"라 말할 수 없음. 단 **AI는 알려진 위협 탐지에 탁월**하고, 알려지지 않은 위협도 이 기본 원리로 접근한다.

## 3. (데이터 한계 — IP/payload 부재)
- 실제 환경에서도 이런 경우 많다: 패킷 위변조, 장비가 로그를 제대로 못 찍음, 프로토콜별 생략 등. **이런 환경에서도 탐지될 수 있어야 하고, 이게 기본이다.** 어렵게 느끼면 안 됨.
- IP 의존적 탐지를 생각하는데, **패킷에 특성들이 있고 그것으로 탐지**한다. 예: **오픈된 보안 탐지 규칙(Snort/Suricata 등)**을 보면 다양한 힌트를 얻을 수 있다.
- **자원 활용은 부수적.** 자원 거의 없이 탐지되는 경우도 있음(상세 생략).

## 4. (위험도 점수)
- 상황에 따라 다르다. **같은 로그도 어떨 땐 위험도가 높고 어떨 땐 낮다.** 이는 **공격 연관 분석** 등 다양한 개념과 함께해야 한다.
- 처음엔 한번 정해보라. **힘을 너무 들이지 말 것. 과탐도, 미탐도 괜찮다. 이 테스트를 많이 해볼수록 도움.**

## 격려 / 운영
- "충실히 쌓이고 있어서, 잘 다듬으면 **소논문**이 나올 수 있겠다. 기대한다."
- 궁금한 점은 **이번 주차 전체 과제 제출 시 종합해서** 보내라. 멘토 매칭 전까지 교수님이 회신.

## 우리에게 주는 재프레이밍 (요약)
1. **"제로데이/처음 보는 공격" 단어 폐기** — 우리 봇넷(3/2)도 *알려진 공격*. 우리 '미관측'은 "학습 분할엔 없지만 알려진 공격"이며, 이걸 "데이터가 달라져도 탐지되나(=규칙 정교화)"로 본 것은 **타당한 평가**(교수님이 1번 답이라 인정).
2. **목표 상향** — 알려진 공격은 ~100% 탐지가 기본. 봇넷 0.49는 천장이 아니라 **행위 분석 정교화 부족** 신호. "전이가 본질적 천장"은 성급한 결론.
3. **방향** — ① 단위 행위 특성의 정교한 분석(행위 기반) ② 시나리오 기반 탐지 ③ 공격 연관 분석 ④ 오픈 규칙(Snort/Suricata)에서 행위 특성 힌트 ⑤ IP 비의존·패킷 특성. **자원 효율은 메인 아님.** 위험도는 가볍게+반복 테스트.
