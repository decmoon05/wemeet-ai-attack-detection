## [ 수행 결과 ]

### 1. 프로젝트 개요
- **프로젝트 수행 과제명:** AI 기반 공격탐지 에이전트 (네트워크 침입 탐지 및 보안로그 분석)
- **과제 선정 배경 및 필요성**
  - ❍ 포트스캔·DoS/DDoS·브루트포스·봇넷 등 공격이 다양화되는데, 규칙 기반 탐지는 신규·변형 공격에 약하고 사람이 로그를 직접 보기엔 양이 방대하다.
  - ❍ ML 기반 탐지가 대안이나 공개 벤치마크(CSE-CIC-IDS2018)는 라벨오류·지름길(shortcut) 특징이 있어 "무작위 평가 고정확도"가 실제 성능을 과대평가한다. 정직한 평가와 해석가능한 결과가 필요하다.

### 2. 목표 및 수행 내용
- **팀 구성**
  - ❍ 2인(팀장: 데이터·모델·평가·문서 / 팀원: 에이전트·시각화·통합). AI 보조 개발 + 검증 라이브러리(scikit-learn·XGBoost·SHAP) 활용.
- **해결할 문제 도출 및 사용자(분석가) 정의**
  - ❍ 보안관제 분석가(SOC)가 대량 트래픽 중 의심 이벤트를 우선 처리해야 하나, 단순 알림은 근거가 없어 판단이 어렵다.
  - ❍ 신규(미관측) 공격은 기존 ML이 거의 못 잡는다 → "어디까지 신뢰 가능한가"를 정량화해야 한다.
- **목표 설정**
  - ❍ 정상/공격 이진 분류 + 0~100 위험점수 + SHAP 근거 + LLM 자연어 설명을 한 시스템으로 통합.
  - ❍ 원칙: 탐지·점수는 ML이 결정, LLM은 판정 변경 없이 설명만(재현성·프롬프트 인젝션 방지).
- **문제 해결 과정**
  - ❍ (발견) 1일치 베이스라인 F1 1.000 → 단일 피처가 중요도 99.2% 차지하는 지름길 허수임을 발견. 평가 설계를 전면 재설계.
  - ❍ (정직한 평가) 다중 일자 결합 + 누수/지름길 피처 제거 + 날짜 교차 평가(과거→미래) → 무작위 0.99 vs 날짜교차 대부분 0(미관측 공격 일반화 붕괴) 확인.
  - ❍ (병목 규명) target-day oracle에서 Bot 0.996 → "표현 부족이 아니라 과거→미래 전이(분포변화)" 문제임을 입증. 보정은 conformal+Neyman-Pearson으로 분포무관 오탐 제어.
  - ❍ (극복) 시간맥락(temporal) 피처(Dst Port별 causal 시간창·반복 규칙성·연결실패율) 추가 → 미관측 공격 탐지율 급상승.
- **수행 결과 및 사용자 검증(의견)**
  - ❍ 미관측 공격 운영 탐지율(오탐 ~15~18/10만 flow 기준): **Bot 0.009→0.49, DoS 0.51→0.62, DDoS 0.38→0.50, BruteForce 0.74→0.98** (Web은 0, 구조적 한계).
  - ❍ SHAP 근거 + 자연어 5섹션 설명 + Streamlit 위험순 triage 대시보드로 분석가가 근거와 함께 우선 처리 가능.
  - ❍ 보안로그(Windows 이벤트) 분석 보조 레이어로 봇넷 지속성 흔적(서비스 설치/변경 EventID) 등 호스트 내부 이상징후 제시 → 다층 방어 개념 실증.

### 3. 활용 방안 및 기대효과
- **기존 방식과의 차별성 및 기대효과**

| 기존 방식 | 차별화 내용(해결) |
|---|---|
| 무작위 분할 고정확도 보고 | ⇨ 날짜 교차 + shortcut 감사로 과대평가 제거 |
| 단일 정확도 제시 | ⇨ 공격군별·노출수준별 정직한 한계 제시 |
| 시간 맥락 없는 flow 단독 | ⇨ causal temporal 피처로 미관측 공격 0→0.49 |
| LLM이 탐지 판정 | ⇨ LLM은 설명만(비권한·인젝션 면역) |

  - ❍ 분석가용 triage(위험점수·근거)로 의심 이벤트 우선 처리. 평가 프로토콜·코드는 후속 보안 실습·연구에 재사용. 단일 ML 한계를 실증해 현업 다층 방어(SIEM+NDR+EDR) 필요성을 데이터로 이해.
- **향후 보완할 사항**
  - ❍ Web(XSS/SQLi): flow에 payload 정보 없어 구조적 미탐 → 패킷/HTTP 레이어 필요.
  - ❍ IP 부재로 host/edge(봇넷 beaconing) 피처 불가 → 원본 PCAP 재처리 시 추가 상승 기대. NDR+로그+EDR 다층화 및 LLM 에이전트 오케스트레이션.
